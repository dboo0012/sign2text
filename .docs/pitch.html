📊 Data Flow Pipeline

1
Video Capture: Browser accesses camera using getUserMedia(), streams 720p video at 30fps to React component
↓
2
Pose Detection: MediaPipe processes video frames (15fps) → extracts 33 body keypoints + 21 hand keypoints per hand + 468 face landmarks
↓
3
Keypoint Processing: Normalize coordinates → smooth temporal data → create sequence buffers (30-60 frames) → compress for transmission
↓
4
WebSocket Transmission: Send JSON keypoint sequences (2-3KB per sequence) to FastAPI backend via Socket.IO
↓
5
Backend Processing: Validate keypoints → preprocess for UniSign → run model inference → get sign prediction + confidence
↓
6
Translation: Look up cached translation → if not found, call Google Translate API → cache result in Redis
↓
7
Response: Send JSON response with predicted sign, translations, confidence scores back through WebSocket
↓
8
UI Update: React components update in real-time showing prediction, translation, and confidence visualization

🔄 Framework Integrations

React ↔ MediaPipe: Direct integration via npm package, runs in browser WebWorkers for performance

React ↔ FastAPI: WebSocket connection using Socket.IO for real-time bidirectional communication

FastAPI ↔ UniSign: Direct Python import, model loaded in memory with GPU acceleration

FastAPI ↔ Redis: Connection pooling for caching translations and session data

⚡ Performance Optimizations

Client-side: WebGL acceleration, frame skipping, keypoint smoothing, sequence buffering

Server-side: Async processing, model caching, batch inference, connection pooling

Network: Data compression, WebSocket keep-alive, intelligent retry logic

Target Metrics: <200ms end-to-end latency, 15fps processing, 95% uptime

🏗️ Deployment Architecture

Frontend: React build served via CDN (Cloudflare/AWS CloudFront)

Backend: FastAPI in Docker containers, load balanced with Nginx

ML Model: GPU-enabled instances (AWS P3/Google Cloud GPU)

Cache: Redis cluster for high availability

🔧 Development Workflow

Local Dev: docker-compose with hot reload for both React and FastAPI

Testing: Jest for React, pytest for FastAPI, WebSocket integration tests

CI/CD: GitHub Actions → Docker builds → automated deployment

Monitoring: Prometheus metrics, Grafana dashboards, error tracking

🎯 Key Integration Points

  • Camera → Pose Estimation: Direct pixel data processing using MediaPipe's JavaScript API
  • Pose Data → WebSocket: JSON serialization of keypoint arrays with compression
  • WebSocket → Model: Async queue processing with ThreadPoolExecutor for CPU-intensive inference
  • Model → Translation: Direct API calls with Redis caching layer
  • Backend → Frontend: Real-time updates via WebSocket with automatic reconnection
